Sentiency

Real-time prompt injection defense

Browser-native protection against hidden and adversarial inputs to AI — before they reach the model.

No backend No telemetry Chrome extension Local-first

Why we built it

Inspiration and problem

Inspiration

A prompt-injection awareness poster in a Google campus hackathon space made it tangible: this is not only a model issue — it is a user safety issue.

Problem

  • Hidden instructions hide in webpages, pasted text, screenshots and diagrams, and live AI sessions.
  • Users often do not know the model is being steered until after the fact.

“We protect users before malicious instructions enter the model context.”

Product

Six detection engines. One shield.

Differentiation

Why Sentiency stands out

  • Browser-native — runs where the user actually works.
  • No backend server — we do not host your traffic.
  • No telemetry — no Sentiency analytics layer.
  • Local-first storage — settings and threat log in the Chrome extension chrome.storage.local API (no Sentiency-hosted database).
  • Multi-surface — text, images, sessions, paste, DOM.
Honest privacy line: the only outbound classification call is to the Google Gemini API (Generative Language API) using the user’s own API key. Not “100% local analysis” — it is local-first, minimal network footprint, no Sentiency servers.

Implementation

Stack, algorithms, disclosure

Application stack

  • JavaScript (ECMAScript) and React 18
  • Google Chrome Manifest Version 3 extension
  • Chrome Extension JavaScript APIs (runtime, storage, side panel, tabs, scripting, context menus)
  • Tailwind CSS, PostCSS, Autoprefixer
  • Webpack 5 and Babel (preset-env, preset-react)
  • Shadow DOM for content-script user interface isolation
  • MutationObserver and TreeWalker Document Object Model APIs
  • Google Generative Language API — REST generateContent
  • chrome.storage.local for settings and threat log

Local detection algorithms

  • Document Object Model text aggregation and visibility heuristics
  • Unicode homoglyph and zero-width character detection
  • Encoding detection, obfuscation hints, and decoding helpers
  • Instruction-pattern keyword and imperative-phrase heuristics
  • Bracket-delimited injection block span collection
  • Span merge, clamp, and augmentation for highlight and surgical remediation
  • Multimodal classification for image-carried visible text (not steganography)
Artificial intelligence disclosure: we used artificial intelligence tools for development iteration and debugging. The shipped product calls Google Gemini for multimodal threat classification. Architecture and integration decisions are ours.

No Sentiency user account — bring your own Google Gemini API key.

Reflection

Challenges · Learned · Next

Challenges

  • Balancing detection sensitivity with false positives.
  • Extension service worker messaging, timing, and Chrome Manifest Version 3 lifecycle constraints.
  • Reliably surfacing hidden or obfuscated text in arbitrary pages.
  • Multi-turn dialogue and image-carried (visible text) attacks.

What we learned

  • Prompt injection is a systems problem — not only a large language model problem.
  • Safety belongs at the interface layer.
  • Layered defense beats a single detector.

What is next

  • Portable Document Format parsing and optical character recognition — dependencies are listed but not yet wired in source; future work only.
  • Additional file types, explainability, policy and enterprise controls.
