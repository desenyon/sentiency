Sentiency 03 / 14

Context

Why we built it

Inspiration

Prompt-injection awareness at a Google campus hackathon made it concrete: user safety, not only a model problem.

Problem

  • Instructions hide in pages, paste, screenshots, diagrams, and live AI sessions.
  • People often notice steering only after the model has already acted.

We protect users before malicious instructions enter model context.

Sentiency 05 / 14

Architecture

One threat pipeline

  1. Input — DOM, clipboard, session, copy, selection, image.
  2. Local heuristics — visibility, Unicode, encoding, instruction patterns.
  3. Google Generative Language API — generateContent, JSON + responseJsonSchema.
  4. Confirmation — thresholds and local corroboration.
  5. Taxonomy, severity, character-level injection spans.
  6. Persist, notify UI, remediate — warn, highlight, sanitize, block.
Hidden text Unicode Encoding Multi-turn Image text
Sentiency 06 / 14

Differentiation

Browser-native · honest privacy

  • Runs in the browser — where users actually work.
  • No Sentiency backend — we do not host your traffic.
  • No telemetry — no Sentiency analytics layer.
  • Local-first storage — chrome.storage.local for settings and threat log.
  • Multi-surface — text, images, sessions, paste, DOM.
Privacy line: the only outbound classification call is Google Gemini (Generative Language API) with the user’s API key. Not “100% local” — local-first, minimal network footprint, no Sentiency servers.

Implementation

Stack and local algorithms

Application stack

  • JavaScript, React 18, Chrome Manifest V3
  • Extension APIs — runtime, storage, side panel, tabs, scripting
  • Tailwind CSS, Webpack 5, Babel
  • Shadow DOM, MutationObserver, TreeWalker
  • Google Generative Language API REST
  • chrome.storage.local for settings and log

Local detection

  • DOM aggregation and visibility heuristics
  • Unicode homoglyphs and zero-width signals
  • Encoding detection and obfuscation hints
  • Instruction-pattern and imperative heuristics
  • Injection span collection, merge, remediation
  • Multimodal visible text in images

Sentiency 13 / 14

Reflection

Challenges · learned · next

Challenges

  • Sensitivity vs false positives; MV3 timing and lifecycle.
  • Hidden and obfuscated text on arbitrary pages.
  • Multi-turn and image-carried (visible text) attacks.

Learned

  • Prompt injection is a systems problem — not only an LLM problem.
  • Safety belongs at the interface layer; layered defense wins.

Next

  • PDF and OCR (deps listed, not wired) · explainability · enterprise controls.
